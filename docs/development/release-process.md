## Purpose

This document describes the expected validation flow before release or milestone work is pushed.

---

## Validation

Run:

- make test-docs
- make test-vdr
- make test

as appropriate for the scope of the change.

---

## Release Checklist

- Tests pass
- Documentation checks pass
- Documentation updated if required
- Commit history reviewed
- Tag created when appropriate
