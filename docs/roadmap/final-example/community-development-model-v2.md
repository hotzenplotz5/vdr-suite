# Community Development Model

## Purpose

VDR-Suite should be structured so that future contributors can work effectively without requiring historical chat logs.

## Documentation First

Every roadmap phase must be documented.

Every architectural decision should be documented using ADRs.

Current status must always be visible in repository documentation.

## AI Assisted Development

Future contributors may use:

- ChatGPT
- Codex
- Gemini
- Claude
- local LLMs

AI usage is acceptable when architecture, tests and documentation remain consistent.

## Mandatory Completion Criteria

A phase is only considered complete when:

- implementation is finished
- tests pass
- architecture documentation is updated
- roadmap documentation is updated
- current status documentation is updated

## Development Workflow

1. review roadmap
2. review architecture
3. implement phase
4. execute tests
5. update documentation
6. commit changes
7. publish status updates

## Long-Term Goal

Create a repository that remains understandable and maintainable for many years.

Knowledge should live inside the repository rather than inside private conversations.
