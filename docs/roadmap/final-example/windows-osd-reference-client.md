# Windows OSD Reference Client

## Purpose

Demonstrate how a modern Windows client can consume VDR-Suite backend services.

## Example Features

- live status display
- recordings overview
- timer overview
- runtime diagnostics
- snapshot-backed data access
- future multi-VDR selection

## Architectural Role

This client is not intended to become the only frontend.

It acts as:

- architecture proof
- API reference
- integration example
- development example

## Vision

The roadmap should end with a visible application that demonstrates the capabilities of the backend.

The Windows OSD Reference Client fulfills that role.
