# Backend Contribution Guide

Purpose:

Keep backend development incremental and reviewable.

Rules:

- Small phases
- One architectural goal per phase
- Tests must pass before merge
- Update current-status documentation
- Update roadmap documentation
- Add ADRs for major decisions

Recommended workflow:

1. Review architecture
2. Implement feature
3. Add tests
4. Run make test
5. Update documentation
6. Commit phase
7. Create milestone tag

Goal:

Allow multiple contributors and AI-assisted development without losing architectural consistency.
